# Anatomy of a Prompt

## Four components of a prompt

Let's talk about how Amazon describes the anatomy of a good prompt. There are basically four components that they talk about: the instructions, the context, the input data, and the output indicator. Let's look at an example here.

## Instructions

So instructions, just telling the generative AI system what you want it to do. In this example, the instructions might be, "Write a short dialogue between two friends about their favorite books."

## Context

Then you wanna provide any relevant context information. In this example, we might say the two friends are sitting in a cozy cafe on a rainy afternoon. Context could also be some sort of external data that you've injected there. So if I had some external database about examples of dialogue between two friends maybe I could go and retrieve some actual information about what that should be and include that as context.

## Input data

Input data, in this example, we're actually providing details about those two friends to work into the results. We're saying friend two loves fantasy novels, especially those with dragons and magic, and friend two prefers mystery no-novels, especially those with clever detectives and unexpected twists. So that's an example of input data. you could actually also provide examples of what you want there as part of the input data. So the input data could go even further and say, okay, for these sets of friends, here's an example of what the short dialogue might look like, and here's another set of friends and what an example of their dialogue might look like, and that can inform the generative Of the AI system of the exact style that you want in the output. So input data can be used a couple of different ways there.

## Output indicator

And finally, the output indicator, how exactly do I want to receive the output of this system? What constraints do I want? in this example, we're saying the dialogue should be two hundred words long and should express each friend's enthusiasm for their favorite genre. So we're giving it some guidance about how long the output should be and what style it should be in there as well.

## Recap

So to recap, the instructions, that's basically the task that you're asking the large language model to do. You're providing a description or instructions for how the model should perform. The context is any external information that can be used to guide the model. That might be additional information about the environment in this case, or it might be information that I retrieve from some external data store that might be relevant to the problem. The input data is input for which you want a response, and the output indicator is the output type or format.
